## 物理效果

以下是最常用的核心积木，控制角色和平台的基本行为。

### 为 ... 添加物理效果
- **参数**：`sprite`（精灵变量）
- **说明**：为指定精灵启用完整的物理系统，包括：
  - 重力下落
  - 斜坡/台阶自动攀爬
  - 地面吸附（零反弹）
  - 按 A 键跳跃（支持 X 段跳，默认单段）
  - 每次跳跃带有轻微屏幕抖动
  - 梯子攀爬（上下键）
  - 蹬墙跳（贴着 wall-jump 图块按 A，弹起后 8 帧内屏蔽左右输入）
  - 冰面滑行
  - 移动平台/电梯跟随
- **注意**：一个精灵只需要添加一次。

### 为 ... 添加无跳跃物理效果
- **参数**：`sprite`（精灵变量）
- **说明**：为精灵添加物理效果，但**不包含跳跃、梯子攀爬、蹬墙跳能力**。适用于 NPC、敌人或不需要玩家控制跳跃的对象。
- **包含**：重力、斜坡、地面吸附、冰面效果、水平碰撞、移动平台跟随。

### 使用物理移动控制 %sprite 速度 %speed
- **参数**：`sprite`（精灵变量）、`speed`（移动速度，默认 `100`）
- **说明**：让插件自动接管该精灵的左右移动控制。调用后：
  - 按 `左`/`右` 键时，精灵速度直接设为 `±speed`
  - 同时按左右键时，保持当前速度不变
  - 地面松手后由摩擦力系统减速；空中松手后 `vx` 直接归零
- **注意**：调用此指令后，**不要再在 `main.ts` 或其他地方自己设置该精灵的 `vx`**，否则会和插件逻辑冲突。
- **示例**：
  ```typescript
  physics.addPhysics(mySprite)
  physics.moveWithPhysics(mySprite, 100)
  ```

### 设置最大跳跃段数为 ...
- **参数**：`number`（段数，最小 `1`，默认 `1`）
- **说明**：设置角色在空中可以额外跳跃的次数。
  - `1`：只能在地面上跳一次（默认）。
  - `2`：二段跳，在地面上跳一次，空中还能再跳一次。
  - `3`：三段跳，依此类推。
- **注意**：落地后会自动重置计数。墙壁跳不消耗跳跃段数，但也会触发屏幕抖动。

### 设置 ... 为路径移动平台 从 ... 到 ... 速度 ...
- **参数**：`sprite`（精灵变量）、`startLoc`（起点图块位置）、`endLoc`（终点图块位置）、`speed`（移动速度，像素/秒）
- **说明**：将精灵设为在两个指定图块位置之间**往返移动**的路径平台。平台会自动沿直线路径移动，到达终点后立即折返。
- **角色行为**：站在平台上的角色会被**锁定在平台中心**，完全跟随平台运动。
- **位置选择**：使用场景中的图块位置选择器直接在地图上点击选取起点和终点。
- **适用场景**：固定路线的电梯、巡逻平台、关卡中的自动运输装置。

## 物理参数

以下积木用于查询或调整物理系统的各项参数，使用频率较低。

### %sprite 在地面上
- **参数**：`sprite`（精灵变量）
- **返回值**：`boolean`
- **说明**：判断指定精灵当前是否站在地面上。返回 `true` 表示在地面上，`false` 表示在空中（跳跃、下落等）。
- **检测方式**：此判断基于插件内部的像素级物理检测，而不是 MakeCode 原生的 `isHittingTile`：
  - 角色站在 tilemap 图块（含斜坡、台阶、半实心平台）上，且 `vy >= 0`（不处于上升状态）→ `true`
  - 角色站在移动平台或自由移动平台上 → `true`
  - 角色处于跳跃上升阶段（`vy < 0`）→ `false`
  - 角色正在下落且脚下无 solid → `false`
- **用途**：用于限制只有站在地面上才能执行某些操作（如攻击、拾取道具、切换动画状态等）。
- **示例**：
  ```typescript
  game.onUpdate(function () {
      if (physics.onGround(mySprite)) {
          // 在地面上时的逻辑
      }
  })
  ```

### 设置最大上台阶高度为 ...
- **参数**：`number`（像素数）
- **说明**：设置角色走上斜坡或台阶时的最大攀爬高度。默认值 `1`。
- **说明**：如果斜坡太陡（超过此值），角色会被挡住。

### 设置跳跃力度为 ...
- **参数**：`number`（跳跃力度，正数即可）
- **说明**：修改按 A 键时的跳跃力度。默认值 `185`（内部自动转为负数向上弹起）。
- **示例**：`physics.jumpVy(250)` 跳得更高；`physics.jumpVy(120)` 跳得更低。

### 设置地面摩擦力保留比例为 ...
- **参数**：`number`（保留比例，范围 `0.0` ~ `1.0`，默认 `0.5`）
- **说明**：设置角色站在普通地面上时，每帧保留的水平速度比例。
  - `0.5`：每帧保留 50%，衰减 50%（较快停下）
  - 数值越小，减速越快；数值越大，滑行越远。
  - 当速度绝对值低于 `5` 时，会自动归零。

### 设置冰面摩擦力保留比例为 ...
- **参数**：`number`（保留比例，范围 `0.0` ~ `1.0`，默认 `0.98`）
- **说明**：设置角色站在冰面上时，每帧保留的水平速度比例。
  - `0.98`：每帧保留 98%，衰减 2%（滑行很远）
  - 当速度绝对值低于 `5` 时，会自动归零。

### 设置重力加速度为 ...
- **参数**：`number`（重力值）
- **说明**：修改全局重力加速度。默认值 `500`。
- **示例**：`physics.gravityAy(800)` 让下落更快；`physics.gravityAy(200)` 让下落更慢（月球重力）。

### 重力
- **返回值**：`number`
- **说明**：返回当前全局重力加速度的值。

### 跳跃力度
- **返回值**：`number`
- **说明**：返回当前跳跃力度的值。

### 最大跳跃段数
- **返回值**：`number`
- **说明**：返回当前设置的最大跳跃段数。

### 最大上台阶高度
- **返回值**：`number`
- **说明**：返回当前最大上台阶高度的值。

### 地面摩擦力保留比例
- **返回值**：`number`
- **说明**：返回当前普通地面的摩擦力保留比例。

### 冰面摩擦力保留比例
- **返回值**：`number`
- **说明**：返回当前冰面的摩擦力保留比例。

### 梯子列表
- **返回值**：`Image[]`
- **说明**：返回通过 `设置梯子图块为` 设置过的图块列表。
- **用途**：通常在其他代码中配合检测使用。

### 冰面列表
- **返回值**：`Image[]`
- **说明**：返回通过 `设置冰面图块为` 设置过的图块列表。

## 图块物理分组

以下指令均接收一个**图块图片列表**作为参数，用于告诉物理引擎哪些图块具有特殊属性。

### 设置半实心图块为 ...
- **参数**：`list`（图块图片列表）
- **说明**：设置可以从下方跳上去的图块（单向平台）。角色站在上面时视为地面，从下方跳上去时可以穿过。
- **注意**：此扩展中半实心图块的实现需配合物理检测逻辑使用。

### 设置忽略碰撞图块为 ...
- **参数**：`list`（图块图片列表）
- **说明**：设置完全不产生碰撞的图块。角色可以自由穿过这些图块，如同空气。

### 设置右侧单向墙图块为 ...
- **参数**：`list`（图块图片列表）
- **说明**：设置只能从**左侧走向右侧**穿过的图块。当角色位于图块左侧时，该图块视为墙壁；当角色位于右侧时，可以自由穿过。

### 设置左侧单向墙图块为 ...
- **参数**：`list`（图块图片列表）
- **说明**：设置只能从**右侧走向左侧**穿过的图块。与右侧单向墙相反。

### 设置梯子图块为 ...
- **参数**：`list`（图块图片列表）
- **说明**：设置可作为梯子攀爬的图块。角色进入梯子区域时：
  - 重力失效
  - 按 `上`/`下` 键可上下移动
  - 按 A 键不会跳跃（处于梯子上时禁用跳跃）

### 设置蹬墙跳图块为 ...
- **参数**：`list`（图块图片列表）
- **说明**：设置可以触发蹬墙跳的图块。当角色侧面贴着这类图块时按 A 键，会向反方向斜上方弹起。

### 设置冰面图块为 ...
- **参数**：`list`（图块图片列表）
- **说明**：设置冰面图块。角色站在冰面上时：
  - 水平速度衰减极慢（`vx *= 0.98`，几乎不减速）
  - 拥有滑行的手感

## 使用流程

### 最小可用示例

```typescript
// 1. 给主角添加物理效果
physics.addPhysics(mySprite)

// 2. （可选）设置特殊图块
physics.setLadders([tilemap`ladder_tile`])
physics.setSemiSolids([tilemap`grass_platform`])

// 3. （可选）创建路径移动平台
let platform = sprites.create(img`...`, SpriteKind.Player)
physics.setAsPathPlatform(platform, tiles.getTileLocation(0, 5), tiles.getTileLocation(5, 5), 60)
```

## 注意事项

1. **精灵种类**：此扩展内部创建了一个新的精灵种类 `SpriteKind.Platform`，请勿与其他扩展冲突。
2. **添加次数**：`add physics` 对一个精灵只需调用一次，重复调用不会报错但无额外效果。
3. **图块尺寸**：所有图块碰撞检测基于 `16x16` 像素图块网格。
4. **重力方向**：此扩展只支持垂直向下的重力，不支持反重力或斜向重力。
5. **跳跃 Vy 值**：跳跃力度使用负数表示向上（MakeCode 坐标系中 Y 轴向下为正）。
6. **平台边缘**：移动平台上，角色走到平台边缘时会自然掉落。
